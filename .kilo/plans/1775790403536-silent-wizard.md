# plan: fix homepage canvas auto-save and cross-device sync

## 1. goal
ensure every stroke, element add/delete/edit on the homepage canvas saves automatically and syncs across **all client types** (appimage, website, apk) via nocobase backend. instant sync on load/save, not real-time collaboration (single user).

## user requirement
> "instant syncing over realtime. realtime isnt necessary as i am the only user. if it saves in each location (appimage, website, and apk) and they all sync back and forth, that would be perfect"

## 2. current state analysis

### what exists
- **indexeddb storage** (`packages/core/src/features/edgeless/storage/canvas-db.ts`):
  - `oplog` store for operation log entries
  - `checkpoints` store for canvas snapshots
  - `drawings` store for metadata
  - data is written locally but **sync to nocobase is broken**

- **sync service** (`packages/core/src/features/edgeless/sync/canvas-sync.ts`):
  - polls every 5 seconds (`sync_interval_ms = 5000`)
  - attempts to sync to `pkm_canvases` nocobase collection
  - **issues identified**:
    1. `recordop()` is called but commented out on line 1607 of edgelesscanvas.tsx
    2. sync service uses `api.request('pkm_canvases', 'create', ...)` which creates a new record every sync instead of updating
    3. no real-time websocket events for canvas sync
    4. no proper loading from server on page init

- **backend** (`packages/backend/server.js`):
  - socket.io configured but no canvas-specific events
  - no canvas persistence endpoints
  - has infrastructure for `data_update` broadcasts

### root causes of "nothing saves"
1. **line 1607 in edgelesscanvas.tsx**: `useedgelessstore.getstate().recordop(op as any)` is commented out - strokes never get recorded to oplog
2. **element mutations** (add/remove/update) don't call `recordop()` at all
3. **sync creates duplicates**: each sync creates a new nocobase record instead of updating the existing one
4. **no initial load from server**: homepage only checks local indexeddb, never pulls from nocobase
5. **no cross-device awareness**: no websocket events to notify other clients of changes

## 3. solution architecture

### data flow (instant sync, not real-time)
```
stroke/element change
    ↓
recordop() → indexeddb oplog + checkpoint (immediate, async)
    ↓ (after stroke complete)
save to nocobase via rest api
    ↓
on page load: fetch latest from nocobase, merge with local (server wins)
```

### storage hierarchy
1. **indexeddb** (local) - immediate persistence, offline buffer
2. **nocobase pkm_canvases** (remote) - source of truth for cross-device sync
3. no websocket/socket needed (single user, instant sync on save/load is sufficient)

## 4. implementation plan

### phase 1: fix immediate persistence (critical)

#### 1.1 uncomment and fix recordop calls in edgelesscanvas.tsx
- **file**: `packages/core/src/features/edgeless/components/edgelesscanvas.tsx`
- line 1607: uncomment `useedgelessstore.getstate().recordop(op as any)`
- ensure stroke data includes proper `targetid` for deduplication

#### 1.2 add recordop for element operations
- **file**: `packages/core/src/features/edgeless/store.ts`
- modify `addelement`, `removeelement`, `updateelement` to call `recordop`:
  ```typescript
  // in addelement
  const op = { type: 'element-add', element: newel, ... }
  void get().recordop(op)
  
  // in removeelement  
  const op = { type: 'element-remove', targetid: id, ... }
  void get().recordop(op)
  
  // in updateelement
  const op = { type: 'element-update', targetid: id, patch, ... }
  void get().recordop(op)
  ```

#### 1.3 ensure checkpoint saves after every stroke
- verify `triggerlivecheckpoint()` is called after stroke completion
- reduce debounce from 500ms to 100ms for faster persistence

### phase 2: fix nocobase sync

#### 2.1 change sync to use upsert pattern
- **file**: `packages/core/src/features/edgeless/sync/canvas-sync.ts`
- current issue: `create` creates new record every sync, causing duplicates
- fix: use `update` with fallback to `create`:
  ```typescript
  // try update first
  const existing = await api.request('pkm_canvases', 'list', {
    filter: { drawingId }
  })
  if (existing.data?.[0]) {
    await api.request('pkm_canvases', 'update', {
      filterByTk: existing.data[0].id,
      data: { content: json.stringify(payload) }
    })
  } else {
    await api.request('pkm_canvases', 'create', { data: ... })
  }
  ```

#### 2.2 store full checkpoint instead of ops
- send complete canvas state (checkpoint) to nocobase
- simpler than oplog replay for cross-device loading
- structure:
  ```typescript
  {
    drawingId,
    clientId,
    timestamp,
    canvas: fabricjs_json,
    elements: edgelesselement[]
  }
  ```

#### 2.3 reduce sync interval from 5s to 2s
- faster cross-device sync feel
- still batched to avoid spamming

### phase 3: load from server on init

#### 3.1 add server fetch to home.tsx
- **file**: `packages/core/src/pages/home.tsx`
- on mount, after ensuring drawingid:
  1. load local indexeddb state
  2. fetch latest from nocobase in parallel
  3. compare timestamps
  4. if server newer: load server state
  5. if local newer: trigger immediate sync

#### 3.2 add load from server to oplog flow
- **file**: `packages/core/src/features/edgeless/storage/canvas-db.ts`
- add `loadfromserver(drawingid)` function:
  ```typescript
  async function loadfromserver(drawingid: string) {
    const res = await api.request('pkm_canvases', 'list', {
      filter: { drawingId },
      sort: ['-updatedAt'],
      pageSize: 1
    })
    return parse(res.data?.[0]?.content)
  }
  ```

#### 3.3 merge local + server state
- compare `timestamp` field
- newer wins
- if server wins: clear local oplog, save server checkpoint

### phase 4: ensure elements persist

#### 4.1 extend drawop type for elements
- **file**: `packages/core/src/features/edgeless/storage/oplog.ts`
- add types (already has some):
  ```typescript
  | { type: 'element-add'; element: edgelesselement }
  | { type: 'element-remove'; targetid: string }
  | { type: 'element-update'; targetid: string; patch: partial<edgelesselement> }
  ```
- note: current `drawop` type already has `path`, `transform`, `delete`, `bitmap-replace`

#### 4.2 add recordop calls to element mutations
- **file**: `packages/core/src/features/edgeless/store.ts`
- `addelement`: call `recordop({ type: 'element-add', element: newel })`
- `removeelement`: call `recordop({ type: 'element-remove', targetid: id })`
- `updateelement`: call `recordop({ type: 'element-update', targetid: id, patch })`

#### 4.3 include elements in checkpoint
- `pkmgetcanvasjson` already returns `{ canvas, elements }`
- ensure checkpoint includes both
- on load: restore both canvas and elements

## 7. risks and mitigations

### risk: conflict when editing same canvas on multiple devices
- **mitigation**: use `updatedat` timestamp, last-write-wins
- server state is source of truth
- on load: compare timestamps, newer wins

### risk: large canvas state causes slow sync
- **mitigation**:
  - store checkpoints (not individual ops) in nocobase
  - compress old strokes into raster checkpoints locally
  - progressive loading (render visible area first)

### risk: network failure during save
- **mitigation**:
  - all ops go to indexeddb first (offline-safe)
  - sync retries with exponential backoff
  - visual indicator when sync pending

### risk: indexeddb quota exceeded
- **mitigation**:
  - prune old ops after checkpoint (already implemented)
  - limit checkpoints to 3 per drawing (already implemented)
  - compress old strokes into raster checkpoints
