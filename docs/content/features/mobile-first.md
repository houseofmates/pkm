# mobile-first

pkm is designed mobile-first — the interface works on small screens before scaling up to desktop.

## why mobile-first

- capture happens everywhere, not just at a desk
- quick review and triage should work on a phone
- reduced friction means more consistent use
- the [[capture-system]] and [[daily-workflow]] are optimized for touch

the mobile app is built with capacitor. see also: [[system-overview]]
